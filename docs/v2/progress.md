## 2026-04-20 — Sub-phase 1c: Cards tabs

Binder + Wishlist ship. `app-v2/api/sync.ts` is a one-line re-export of v1's `api/sync.ts` (both `handleWants` and `handleAvailable` are exported there); v2's vercel.json rewrites `/api/sync/wants` and `/api/sync/available` onto it. Zero duplication; v1's patches automatically flow. The card catalog is shared via a relative symlink `app-v2/public/data -> ../../public/data` — Vite copied all 29 JSON files through the symlink into `dist/data/` at build, so v2's CDN serves the same catalog URLs v1 does. v1's 2-hour refresh cron (GitHub Actions → deploy-hook) keeps the files fresh without v2-side plumbing. Three new hooks: `useProductIndex` + `useFamilyIndex` (static JSON, cached forever in React Query); `useBinder` + `useWishlist` (React Query + optimistic PUT mutations with rollback on error, invalidate on settle). `CardPicker` Sheet loads `jump-to-lightspeed.json` as the Phase-1 search pool and accepts a search substring + tap-to-pick; the parent Cards route routes the pick to binder (by productId) or wishlist (by familyId computed via the same slug rule v1 uses). Binder rows join against product-index for display name + set; Wishlist rows join against family-index for name and render "Any variant" chip (variant restrictions ship later — Phase 2 if beta feedback demands). Priority star toggles via a 44×44 leading button, renders a visible gold star when set, and sorts priority to the top of the list. Ghost + signed-out users see the same `EmptyState` with the gold "Continue with Discord" CTA per design §4.5/§4.6 — no server write attempted, matching the merge-path invariant. Dev-server caveat: `vite dev` has no `/api/*` endpoints, so useAuth always shows signed-out under `npm run dev`; full auth flow needs `vercel dev`. Smoke tests still green (4 passed). Bundle at 138KB gzipped (up 3KB from 1b — the real work is Framer + Radix from earlier). Next up: 1d trade canvas — `/s/:code` against a server-backed open session on tap-one, balance strip, tap-to-expand PriceSheet.

## 2026-04-20 — Sub-phase 1d: Trade canvas (solo)

The universal trade surface lands. `app-v2/api/sessions.ts` re-exports v1's full nine-action dispatcher (get / list / create / edit / confirm / cancel / create-open / claim / invite-handle); v2's vercel.json picks up all the pretty-URL rewrites. Tapping the FAB on Trades calls `POST /api/sessions/create-open` (eager session creation per design §5.2 — mints a ghost if needed, sets cookie, returns the new code) and then `navigate('/s/:code')` to mount the canvas. If create-open fails (the canonical LGS bad-cell case from §10 1d Obs 3), the Trades route surfaces a `role="alert"` banner with inline Retry instead of navigating — no offline queue, no client-only draft. Playwright smoke covers this path directly against `vite dev` (which has no `/api/*`, so the mutation reliably fails). On the canvas itself, `useSession(code)` is a React Query wrapper around `GET /api/sessions/:id` with optimistic edit/confirm/cancel/claim mutations; status cascades through `pending → preview | ready | not-found | error`. The canvas renders: a state chip (`Live` / `Waiting for partner` / `Settled` / `Cancelled` / `Expired` — mapped to the state-tone palette from §6.1), a `TradeSide` panel for "You offer" (editable, side-yours tint behind the label), a `BalanceStrip` with totals + diff + missing-price caption, a `TradeSide` for "They offer" (read-only in Phase 1d, side-theirs tint), and an `ActionBar` whose primary action depends on session state — `Invite someone` for open-slot, `Confirm trade` / `Waiting on @partner` for shared, nothing for terminal. Share button in the NavBar's trailing slot uses `navigator.share` when available and falls back to clipboard + a "Link copied" toast. CardPicker reuses the 1c Sheet primitive. Non-participants visiting a session URL see the preview with a `Join this trade` CTA that calls the claim mutation. `TradeCardSnapshot` + `SessionView` types live in `src/lib/trade.ts`; `totalOf` and `missingPriceCount` helpers pure-functions. Deferred in this commit: `PriceSheet` (tap-to-expand pricing, market/low toggle, percentage presets) — will land in a follow-up 1d commit. Canvas bundle is +3KB gzipped (138 → 141). Smoke suite expanded to 5 tests (shell + FAB-retry + canvas-not-found + cards-ghost), all green in ~4s. Next up: finish 1d with PriceSheet, then 1e live trade (QR, polling, claim).
