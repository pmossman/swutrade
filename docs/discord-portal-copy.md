## OAuth2 tab

### General

- **Redirect URLs** (must include both production and beta):
  - `https://swutrade.com/api/auth/callback`
  - `https://beta.swutrade.com/api/auth/callback`
- **Default Authorization Link** → `Custom URL` (we build it server-side in `api/auth.ts::buildBotInstallUrl`):
  - **Scopes:** `bot`, `applications.commands`, `identify`, `guilds`
  - **Bot Permissions** (numeric: `360777255952`):
    - View Channels (1024)
    - Send Messages (2048)
    - Embed Links (16384)
    - Attach Files (32768)
    - Read Message History (65536)
    - Manage Channels (16) — auto-creates `#swutrade-threads` on install
    - Manage Threads (1<<34)
    - Create Private Threads (1<<36)
    - Send Messages in Threads (1<<38)

### Public install URL

The `Add to Server` button on swutrade.com is rendered server-side via `buildBotInstallUrl()`. The same URL pattern, with `client_id` substituted, is what you'd advertise externally:

```
https://discord.com/oauth2/authorize?client_id=<APP_ID>&scope=bot+applications.commands&permissions=360777255952
```

## Bot tab

- **Username** (the bot user's @handle): `SWUTrade`
- **Privileged Gateway Intents:** all OFF. SWUTrade does not maintain a websocket connection — it's an HTTP-interaction bot. Privileged intents would only be needed for things like reading message content from non-DM channels, which we never do.
- **Public Bot:** ON (allows other server admins to install).
- **Requires OAuth2 Code Grant:** OFF.
